# deliver-with-subagents

[中文](./README.md) | [English](./README_EN.md)

一个面向 Codex 的上下文隔离开发交付 Skill。它把实现、测试和修复交给唯一的 Dev 子会话，把最终 Review 交给独立只读会话，让主会话只保留需求边界、关键决策和可核验结果。

## 使用边界与推荐搭配

本 Skill **只用于需求已经确认后的开发交付阶段**，包括实现、测试、独立 Review、修复和复查；不用于需求发现、方案讨论或范围澄清。

开始开发前，建议搭配需求与规划工具：

- 使用 **OpenSpec** 沉淀需求、验收标准、设计决策和可执行 tasks。
- 对大型、跨阶段或需要多个会话推进的工作，先使用 **Wayfinder** 明确路线、依赖和阶段边界。
- 需求和范围确认后，再调用 `$deliver-with-subagents` 完成交付闭环。

如果仍有关键需求未决，应先返回 OpenSpec、Wayfinder 或其他需求讨论流程，不把猜测带入生产代码。

## 安装与调用

将仓库放到 Codex 可发现的 Skill 目录：

```bash
mkdir -p ~/.agents/skills
git clone https://github.com/lcw363/deliver-with-subagents.git \
  ~/.agents/skills/deliver-with-subagents
```

在任务中显式调用：

```text
请使用 $deliver-with-subagents 完成 openspec/changes/add-order-refund，主会话只保留关键结论。
```

安装后，Codex 也可以在任务符合 Skill 描述时自动选择它；显式写出 `$deliver-with-subagents` 更容易固定执行方式。

## 交付流程

1. **锁定需求**：读取用户指令、OpenSpec/PRD/tickets 和项目规则，明确验收标准、不做内容、测试策略及发布边界；影响业务、数据、权限、安全或 API 契约且仍无法判断的问题，暂停并请求确认。
2. **开发**：启动唯一一个 Dev 子会话作为代码写入者，完成最小生产可用实现；随后简化本次改动并补充必要的关键注释。
3. **测试**：先运行最窄相关测试，再运行受影响模块的单元、集成、类型检查、编译或构建。只有行为可测试且公共测试缝清晰时采用 TDD；Bug 修复优先先写回归测试。
4. **真实接口验证**：新增或修改 HTTP API 时，除自动化测试外，还应启动本地服务或使用已授权测试环境，以脱敏后的具体实际参数调用真实路由，验证 HTTP 状态、响应字段、业务结果、数据副作用以及清理或回滚结果。缺少服务、凭据或有效 fixture 时必须标记 `BLOCKED`，不能用 Mock 或 schema 检查冒充真实接口验收。
5. **独立 Review**：最终 Review 不在 Dev 子会话中完成。已提交分支使用 `$code-review`；未提交改动使用专用未提交 Review，或启动只读 Review 子会话。只返回带等级、文件位置、证据、影响和最小修复建议的可执行问题。
6. **修复与复查**：把成立的问题交回原 Dev 子会话做最小修复、补回归测试，再进行独立 Review。循环到没有已知且可执行的 P0-P3 问题，或达到适用时限。
7. **压缩汇报**：主会话只汇总需求完成情况、关键文件、测试和真实接口证据、Review/修复结果、P0-P3 状态、未完成项与发布状态。

## 关键限制

- **单写者**：任何时刻只有 Dev 子会话可以修改代码；Review 子会话保持只读，避免并发写入冲突。
- **条件性 TDD**：不机械强制先写测试；测试缝明确时使用 TDD，Bug 修复优先先补复现测试。若测试缝选择会改变设计，则先确认。
- **发布默认关闭**：除非用户明确要求，不执行 commit、push、PR、部署或 OpenSpec archive。
- **问题必须闭环**：P0-P3 均指存在具体失败路径、能够执行修复的问题；纯风格偏好不算 P3。Review 发现的问题应修复并复查，不能含糊略过。
- **可配置收口时限**：优先使用用户指定的时长；用户未指定时，从第一版实现完成后默认计时 60 分钟。用户可明确要求其他时长或不限时。达到适用时限后立即停止修改，并列出未完成项、未验证项、阻塞和残余风险；不限时也不能跳过用户确认、权限、安全或环境边界。
- **真实测试有边界**：Token 通过环境变量提供，凭据和个人信息必须脱敏；64 位 ID 在 JSON 中按字符串处理；未经明确授权不访问生产环境或使用不可回滚的生产数据。

## 简短示例

```text
请使用 $deliver-with-subagents 实现 OpenSpec change `add-order-refund`：
- 只实现 tasks.md 中 Phase 1；
- 新增退款接口后，用测试商户和订单号调用本地真实 HTTP 路由；
- Dev 子会话负责实现、简化、测试和修复，独立会话负责 Review；
- 主会话只保留关键决策、测试证据和 P0-P3 结果；
- Review、修复和复查的收尾时限为 90 分钟；
- 不提交、不推送、不部署；不确定且影响退款金额或权限时停下等我确认。
```

完整执行规则见 [SKILL.md](./SKILL.md)。
